## Document Organization and Folders

Create a logical structure for your documentation using folders and subfolders. You nest pages deeply to match your project's architecture, such as `/api/reference` or `/guides/advanced`.

<Steps>
  <Step title="Create a Folder" icon="folder-plus">
    Navigate to your workspace root. Click the new folder icon and name it, e.g., `features`.
  </Step>
  <Step title="Add Documents" icon="file-plus">
    Inside the folder, create new MDX pages. Drag and drop to reorganize.
  </Step>
  <Step title="Manage Permissions" icon="shield">
    Set read/write access per folder for team members.
  </Step>
</Steps>

<Callout kind="tip">
  Use consistent naming conventions like kebab-case for folders to improve navigation.
</Callout>

## Version Control for Docs

Every edit creates a new version automatically. You view diffs, restore previous states, or branch docs for experiments.

```javascript
// Example: Fetch version history via API
const response = await fetch('https://api.example.com/docs/my-page/versions', {
  headers: { Authorization: `Bearer ${YOUR_TOKEN}` }
});
const versions = await response.json();
console.log(versions); // Array of { id, date, author, changes }
```

<Tabs>
  <Tab title="Restore Version" icon="history">
    Select a page, click History, and choose a version to revert.
  </Tab>
  <Tab title="Compare Changes" icon="git-compare">
    View side-by-side diffs highlighting additions and deletions.
  </Tab>
</Tabs>

## Collaboration Tools for Teams

Invite team members to co-edit docs in real-time. Use comments, @mentions, and approvals for workflows.

<CodeGroup tabs="Invite,Comment">
```javascript
// Invite user via API
await fetch('https://api.example.com/teams/invite', {
  method: 'POST',
  body: JSON.stringify({ email: 'team@company.com', role: 'editor' })
});
```
````javascript
// Add inline comment
await fetch('https://api.example.com/docs/my-page/comments', {
  method: 'POST',
  body: JSON.stringify({
    content: 'Consider adding examples here',
    position: { line: 42 }
  })
});
````
</CodeGroup>

<Expandable title="Advanced Workflow" default-open="false">
  Set up approval gates: Draft → Review → Published. Assign reviewers per doc type.
</Expandable>

## Search and Filtering Options

Powerful full-text search indexes all content. Filter by tags, authors, dates, or folders.

| Filter Type | Description | Example |
|-------------|-------------|---------|
| Tags | Match doc labels | `api`, `v2.0` |
| Authors | By contributor | `john.doe@company.com` |
| Date Range | Recent changes | `>2024-01-01` |
| Folders | Scoped results | `/guides/` |

<Callout kind="info">
  Search supports fuzzy matching and regex for precise queries like `/error.*handling/i`.
</Callout>

## Export and Sharing Functionalities

Generate PDFs, HTML sites, or ZIP archives. Share public links with password protection or embed previews.

<Steps>
  <Step title="Export to PDF" icon="file-down">
    Select pages or folders, choose PDF, and download.
  </Step>
  <Step title="Create Share Link" icon="link-2">
    Toggle public access, set expiration (e.g., 30 days), and copy URL.
  </Step>
  <Step title="Embed in Site" icon="code">
    Use `<iframe src="https://docs.example.com/embed/my-page" />`.
  </Step>
</Steps>
